Here are some steps to get you started for the hackathon. These should be done before the hackathon begins.

#### 1. Hackathon python environment and notebooks

In order to be able to run the notebooks provided for the hackathon and conduct data analysis using the tools for the hackathon,
the official global hackathon software stack needs to be installed.
Please follow [these instructions](https://github.com/digital-earths-UK-hackathon/hk26/tree/main) to get yourself set up with a working Python environment on JASMIN (or locally).

There are notebooks that give examples of how to access the data, and links to take you to support pages.

#### 2. Further reading

If you would like to learn more about the healpix format in which all the model data is stored please follow this [link](https://easy.gems.dkrz.de/Processing/healpix/index.html).

If you want to see a high-level overview of last year's WCRP global, km-scale hackathon, read this [BAMS article](https://doi.org/10.1175/BAMS-D-25-0183.1).
There is an [in-preperation GMD article](https://digital-earths-global-hackathon.github.io/gmd-paper/) which goes into more detail on the software stack.
